# AI-Powered MCQ Quiz Platform

This document outlines the architecture and implementation plan for a responsive, React-based web application that uses the Gemini API to generate 25-question Multiple Choice Question (MCQ) tests based on user prompts.

## User Review Required

> [!IMPORTANT]
> **Data Storage**: Since no backend server was specified, this application will use the browser's `localStorage` to save all generated tests, user submissions, and analytics. This means the data will only be available on the specific browser and device where the test was taken. Is this acceptable?
>
> **Technology Stack**: We will build this using React with Vite for a fast and modern frontend. For styling, as per guidelines, we will use custom vanilla CSS with premium, dynamic designs (glassmorphism, modern typography, animations) instead of a framework like Tailwind. Let me know if you prefer otherwise!
>
> **API Key Security**: Using the Gemini API key directly in a frontend React app means the key will be visible to anyone inspecting the network tab. For a production environment, this should ideally be routed through a backend. For this project, we will use it directly on the frontend as provided.

## Proposed Changes

We will create a new Vite + React application. The project will be structured into several pages using `react-router-dom`.

---

### Application Setup & Dependencies

We will initialize a Vite project and install the following dependencies:
- `react-router-dom` (for routing between Home, Dashboard, and Reports)
- `@google/generative-ai` (for the Gemini API integration)
- `lucide-react` (for modern, scalable icons)
- `recharts` (for the weak-area analysis visualization)

---

### Core Components and Pages

#### 1. Home Page (`/`)
- A visually stunning hero section.
- A textarea for entering the prompt with character limits (e.g., 50 to 3000 characters).
- Graceful error handling for input limits (resolving the character size error requirement).
- A "Generate Test" button that triggers a loading state while fetching from the Gemini API.

#### 2. Test Taking Interface (`/test/:id`)
- An interactive, responsive interface to take the generated 25 MCQ test.
- Users can select options and submit the test.
- Navigation warnings if trying to leave before submitting.

#### 3. Dashboard (`/history`)
- A grid/list displaying previously taken tests.
- Shows basic stats: Date taken, prompt/topic, and score.
- Clicking on a test navigates to the detailed report.

#### 4. Detailed Report & Analysis (`/report/:id`)
- **Review Section**: Displays each question, what the user answered, and the correct answer. For wrongly answered questions, it provides a detailed explanation generated by Gemini.
- **View Analysis**: A dedicated section/button that visualizes the user's performance across different sub-topics (extracted by Gemini during question generation), highlighting weak areas visually using a chart.

---

### API & Data Structure

When calling the Gemini API, we will use a structured prompt to ensure we get exactly 25 questions in a consistent JSON format.

```json
{
  "topic": "Main topic name",
  "questions": [
    {
      "question": "Question text...",
      "options": ["Option A", "Option B", "Option C", "Option D"],
      "correctOptionIndex": 1,
      "explanation": "Explanation for the correct answer...",
      "subTopic": "Specific sub-topic for analysis"
    }
  ]
}
```

### Aesthetics & Responsive Design

- **Typography**: Using Google Fonts like 'Outfit' or 'Inter'.
- **Color Palette**: A modern dark mode default with deep blues/purples, neon accents, and soft glassmorphism overlays.
- **Animations**: Subtle hover states, smooth page transitions, and loading spinners.
- **Responsiveness**: CSS Grid and Flexbox will ensure perfect rendering on mobile phones, tablets, laptops, and large monitors.

## Verification Plan

### Automated/Functional Testing
- Verify that character limits and input validation work correctly on the Home Page.
- Ensure the application handles Gemini API JSON parsing errors or timeout errors gracefully.
- Verify that navigating across mobile, tablet, and desktop viewports maintains UI integrity.

### Manual Verification
- Generate a test and confirm exactly 25 questions are produced.
- Complete a test, select some wrong answers on purpose, and verify that the report page shows explanations correctly.
- Click "View Analysis" and verify the visual chart accurately reflects the wrong answers' sub-topics.
